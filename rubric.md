# Judging Rubric - Slack Agent Builder Challenge

* **Creativity (33.3%)**: How unique, innovative, and original is the idea? Does it use Slack's interface (DMs, channels, canvases) in a new way?
* **Functionality (33.3%)**: How well does the agent work? Is it stable, interactive, and responsive?
* **Impact (33.3%)**: What value does the agent provide? Does it address a real-world social impact problem (e.g., healthcare accessibility)?

---

## Key Developer Patterns (Judge-Suggested)
* **Human-in-the-loop**: Include explicit approvals for critical database/email changes.
* **Context Awareness**: Work directly in standard channels or canvases.
* **Reliability & Error Handling**: Gracefully handle failed LLM queries or invalid commands.
